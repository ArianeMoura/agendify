# Registro de Contribuição

## Visão Geral

Este documento apresenta, de forma detalhada, as etapas, atividades e responsabilidades da equipe no desenvolvimento do projeto **Agendify**. A construção do sistema foi organizada em cinco etapas principais, cada uma contendo tarefas específicas, prazos definidos e artefatos entregues.

## Etapa 1: Concepção, Proposta de Solução e Início da Elaboração do Projeto da Solução

### Objetivo

Formar a equipe, definir o problema a ser trabalhado, e iniciar o projeto da solução tecnológica. Esta etapa foca em criar a documentação de contexto (visão geral, problema, objetivos, justificativa, público-alvo) e a especificação do projeto (personas, histórias de usuários, requisitos, arquitetura distribuída, planejamento e rastreabilidade).

### Tarefas

- **Reunião com stakeholders**: Realizar reuniões com as partes interessadas para entender as necessidades e expectativas.
- **Levantamento de requisitos**: Documentar os requisitos funcionais e não funcionais do projeto.
- **Análise de viabilidade**: Avaliar a viabilidade técnica e econômica do projeto.
- Documento de Contexto.
- Especificação do Problema.
- Documentação dos perfis de usuários, suas necessidades e Personas.
- Documentação da Arquitetura Distribuída.
- Documentação dos requisitos funcionais e não funcionais do projeto.

### Prazo

- **Data de conclusão**: 07/09/2025

---

## Etapa 2: Desenvolvimento do Back-end: Web API

### Objetivo

Desenvolver o Back-end da aplicação distribuída, implementando uma Web API Rest, configurando o ambiente, modelando os dados e implementando as funcionalidades de banco de dados NoSQL para armazenamento e recuperação de dados, incluindo autenticação, recursos e rotas.

### Tarefas

- **Desenvolvimento da API**: Implementar as APIs necessárias para o funcionamento do sistema.
- **Gestão de Banco de Dados**: Projetar e implementar o banco de dados.
- **Testes Unitários**: Criar e executar testes unitários para garantir a qualidade do código backend.
- Documentação da arquitetura da aplicação distribuída.
- Projeto da arquitetura baseada em API.
- Implementação da API e testes de unidade.
- Documentação com justificativa e avaliação do modelo de dados NoSQL.
- Implementação das funcionalidades que envolvem o banco de dados NoSQL.
- Construção da Arquitetura de Solução.

### Tarefas Específicas

- Construção do Banco de Dados MongoDB e API.
- Criação e implementação das funcionalidades de Gerenciar Espaços e seus recursos, incluindo a camada de Controller, os Models e a Service responsável pela lógica de negócio.
- Construção da Arquitetura de Solução, Diagrama de Classes, Esquema e Documentação do Banco de Dados.
- Criação e implementação das funcionalidades relacionadas à Reserva, abrangendo a camada de Controller, os Models e a Service responsável pela lógica de negócio.
- Documentação das Tecnologias Utilizadas e suas Justificativas, assim como Qualidade de Software.

### Prazo

- **Data de conclusão**: 05/10/2025

---

## Etapa 3: Desenvolvimento do Front-end: Web

### Objetivo

Desenvolver o primeiro Front-end do projeto (Web), trabalhando com modelagem de processos (BPMN), projeto de interfaces (wireframes), fluxogramas de interação, e codificação. Os dados apresentados serão acessados via Web API. O desenvolvimento será submetido a testes de unidade e integração.

### Tarefas

- **Desenvolvimento Frontend**: Criar a interface de usuário utilizando as tecnologias web escolhidas.
- **Integração com Backend**: Integrar o frontend com as APIs desenvolvidas na etapa anterior.
- **Testes de Interface**: Realizar testes de usabilidade e interface para assegurar a qualidade da aplicação web.
- Documentação do projeto de interface e do fluxo de telas do sistema (aplicação web).
- Implementação do projeto de interface e fluxo de telas (aplicação web).
- Implementação e documentação dos testes de integração.
- Relatórios de gestão: "retrato" do GitHub Projects e das contribuições.

### Tarefas Específicas

- Construção e inicialização do repositório frontend web, desenvolvendo e implementando a Tela de Login, incluindo a integração com a API para autenticação do usuário e Tela de gerenciamento de Espaços.
- Elaboração da documentação da Modelagem do Processo de Negócio e Indicadores de Desempenho.
- Desenvolvimento e implementação da Tela de Gerenciamento de Reservas (Administrador) e da Tela de Reservas (Usuários).
- Criação e documentação do Projeto de Interface e do Template Padrão da Aplicação, incluindo definição do design, logo, identidade visual, tipografia e elaboração dos wireframes das telas.
- Desenvolvimento da Tela de Perfil e Dashboard.
- Criação do Diagrama de Casos de Uso.

### Prazo

- **Data de conclusão**: 26/10/2025

---

## Etapa 4: Desenvolvimento do Front-end: Mobile

### Objetivo

Desenvolver o segundo Front-end do projeto (Mobile), adaptando a modelagem de processos, projeto de interfaces (wireframes) e fluxos de interação para dispositivos móveis. Os dados também serão acessados via Web API. O desenvolvimento será submetido a testes de integração e sistema.

### Tarefas

- **Desenvolvimento Mobile**: Implementar a versão mobile do projeto para Android e iOS.
- **Integração com Backend**: Garantir que a aplicação mobile se comunique corretamente com o backend.
- **Testes em Dispositivos**: Realizar testes em diferentes dispositivos móveis para assegurar compatibilidade e desempenho.
- Documentação do projeto de interface e fluxo de telas (aplicação mobile).
- Implementação do projeto de interface e fluxo de telas (aplicação mobile).
- Implementação e documentação dos testes de sistema.
- Relatórios de gestão: "retrato" do GitHub Projects e das contribuições.

### Tarefas Específicas

- Responsável pela hospedagem e configuração do ambiente do projeto.
- Desenvolvimento e implementação das telas mobile, incluindo a Tela de Login, Tela Inicial e a Tela de Editar Perfil.
- Elaboração, execução e documentação dos testes do sistema.
- Desenvolvimento das funcionalidades e telas mobile relacionadas às Reservas, incluindo edição, exclusão, modal de Editar Reserva e modal de Nova Reserva.
- Desenvolvimento das funcionalidades e tela mobile de Gerenciamento de Espaços.
- Criação e documentação dos Diagramas de Fluxo do sistema e Registro de Contribuição.
- Realização de ajustes e atualização da documentação geral do projeto.
- Criação das funcionalidades e da Tela de Relatório e Gráfico do Sistema (Analytics).
- Criação da Tela de Recuperação de Senha e desenvolvimento das funcionalidades de notificações de reservas e da funcionalidade de avaliação de espaços, serviços, visualização das avaliações e agradecimento por avaliar.

### Prazo

- **Data de conclusão**: 16/11/2025

---

## Etapa 5: Diagnóstico, Entrega da Solução e Apresentação

### Objetivo

Finalizar o ciclo de desenvolvimento do projeto através da consolidação de todos os artefatos e da solução tecnológica (Web, Mobile e API). Esta etapa foca na análise crítica do processo, na avaliação das tecnologias utilizadas, na produção do documento de "Considerações Finais" e na preparação da apresentação final (vídeo e slides) do projeto.

### Tarefas

- **Homologação**: Realizar testes finais para validar a solução.
- **Correção de Bugs**: Corrigir quaisquer problemas encontrados durante a homologação.
- **Revisão Final:** Correção de artefatos e bugs remanescentes das etapas anteriores.
- **Documentação Final:** Elaboração do documento "Considerações Finais", contendo análise crítica, avaliação de frameworks e propostas de melhoria.
- **Diagnóstico de Gestão:** Atualização final do GitHub Projects e geração dos gráficos de contribuição ("Retrato" do time).
- **Apresentação:** Criação dos slides, roteiro de apresentação e gravação do vídeo demonstrativo da solução em funcionamento.

### Prazo

- **Data de conclusão**: 07/12/2025

## Comunicação e Colaboração

### Ferramentas de Comunicação

- **GitHub:** Para versionamento de código e revisão de pull requests.
- **GitHub Projects:** Para gerenciamento de tarefas e acompanhamento do progresso.
- **WhatsApp:** Para comunicação diária e rápida.
- **Microsoft Teams:** Para reuniões e discussões detalhadas.

### Reuniões Regulares

- **Reunião semanal**: Toda segunda-feira às 22h.
- **Reunião de revisão**: Sempre que necessário e ao final de cada etapa para revisar o progresso e ajustar o plano.
