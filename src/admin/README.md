# Agendify Admin (Next.js)

Administrative web panel for Agendify — the **Org Admin** experience: manage spaces, bookings,
reports, and user access. Consumes the [Agendify API](../../README.md).

## Prerequisites

Node.js 20+ (`.nvmrc`), and a reachable Agendify API.

## Setup

```bash
npm ci
cp .env.example .env.local   # aponte para a API (Render por padrão, ou localhost)
npm run dev                  # http://localhost:3000
```

To run fully local instead of against Render, start the API (`cd ../api && dotnet run`) and set
`NEXT_PUBLIC_API_URL=http://localhost:5089` in `.env.local`.

## Environment

| Variable              | Purpose                                                                        |
| :-------------------- | :----------------------------------------------------------------------------- |
| `NEXT_PUBLIC_API_URL` | API root URL (no `/api`). Default `http://localhost:5089`. See `.env.example`. |

## Stack

Next.js 16 (App Router) · React 19 · TypeScript (strict) · Tailwind CSS v4 · TanStack Query.

- **Design system** — brand tokens (light + dark) in `app/globals.css` via Tailwind v4
  `@theme inline`; theme switching with `next-themes`. Fonts **Sora** + **Manrope** self-hosted via
  `next/font`. See [Design System](../../docs/DESIGN-SYSTEM.md).
- **Components** — accessible library in `components/ui/` built on Radix primitives (Dialog,
  Dropdown, Tooltip, Tabs, Switch, Avatar) + `Field` (label↔control), toasts via `sonner`.
- **Brand** — icon/wordmark/lockup in `components/brand/`; favicons/PWA generated by
  `scripts/generate-icons.mjs` (`sharp`).
- **Layout** — responsive shell in `components/layout/` (sidebar + mobile drawer, theme toggle,
  skip-link, `aria-current`).

## Accessibility

Targets WCAG 2.2 AA: semantic HTML, associated form labels, keyboard-operable Radix primitives,
visible focus, skip-link, `prefers-reduced-motion`. `jsx-a11y` runs in lint; `vitest-axe` asserts
zero violations on key components.

## Security

Security headers + CSP set in `next.config.ts` (relaxed in dev for HMR). Auth tokens live in
`localStorage` with silent refresh on 401 (`lib/api.ts`, `lib/auth.tsx`).

> Known trade-off: `localStorage` tokens are readable by XSS. The hardened path is httpOnly,
> `SameSite` refresh cookies — this requires an API/backend change and is tracked as a follow-up.
